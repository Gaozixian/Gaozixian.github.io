CBAM-LSTM: 端到端自动驾驶的时空注意力网络

CBAM-LSTM: 用于端到端自动驾驶的混合时空注意力网络

结合时序建模(LSTM)和空间-通道注意力(CBAM)提升自动驾驶模型的感知与决策能力

摘要与核心创新

研究动机

端到端自动驾驶需要同时处理时序动态(如车辆轨迹)和空间特征(如道路结构)。现有方法中，CNN缺乏时序建模能力，而LSTM忽视了空间注意力机制。

核心创新
  • 双分支融合架构：CBAM(空间+通道注意力) + LSTM(时序建模)
  • 轻量化设计：CBAM嵌入CNN骨干网络(如ResNet-18)，减少计算开销
  • 时空注意力可视化：展示模型在复杂场景下的关注区域

论文主体框架

1. Introduction

  • 背景与挑战：空间特征丢失(如交通标志)、时序决策不连贯(如转弯时的速度调整)
  • 现有方法局限：CNN缺乏时序建模，LSTM忽视空间注意力
  • 创新点：CBAM-LSTM的混合架构与轻量化设计

2. Related Work

3. Methodology

整体架构图应包含以下模块：

  • 输入：多帧图像(时序) + 传感器数据(可选)
  • CBAM-CNN分支：提取空间特征并加权重要区域
  • LSTM分支：编码历史动作和当前特征，输出控制指令
  • 融合策略：时空特征拼接或交叉注意力
class CBAM_LSTM(nn.Module):
    def __init__(self):
        self.cnn = ResNet18(with_cbam=True) # 插入CBAM的ResNet
        self.lstm = nn.LSTM(input_size=cnn_feat_dim + state_dim, hidden_size=128)
        self.head = nn.Linear(128, 3) # 输出：方向盘、油门、刹车

    def forward(self, x_frames, x_states):
        # x_frames: [T, C, H, W], x_states: [T, state_dim]
        spatial_feats = [self.cnn(frame) for frame in x_frames] # CBAM加权
        lstm_input = torch.cat([spatial_feats, x_states], dim=-1)
        controls, _ = self.lstm(lstm_input) # 时序建模
        return self.head(controls[-1]) # 最后一步预测

4. Experiments

  • 数据集：CARLA(仿真)、UDACITY(真实数据)
  • 对比模型：Pure CNN、Pure LSTM、CILRS等
  • 评估指标：控制指令误差(MAE)、安全性(碰撞率)、泛化性
  • 消融实验：CBAM的作用、时序长度对LSTM性能的影响

5. Conclusion & Future Work

  • 总结：CBAM-LSTM在复杂场景(如雨天、夜间)的优势
  • 展望：引入多模态(激光雷达)、在线学习机制
